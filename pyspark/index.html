Apache Spark

  • Poderoso motor (engine) voltado ao processamento e análise de dados em grande escala
  • Fornece interface para programação de clusters com paralelismo de dados implícito e tolerância a falhas
  • Originalmente desenvolvido no AMPLab da Universidade da Califórnia (2009), Berkeley
  • Código base doado à Apache Software Foundation (2014)
  • Proposto para simplificar o trabalho com Hadoop e evitar gargalos de desempenho
  • Persiste até 100 vezes mais rápido que o MapReduce tradicional do Hadoop

Linha do Tempo

Arquitetura de uma Aplicação

👨‍💼 Driver Program

Contém a função main() e cria o SparkContext

🎯 Cluster Manager

Gerencia recursos do cluster (YARN, Mesos, Kubernetes)

⚡ Worker Nodes

Executam as tarefas (tasks) em Executors

Fluxo de Execução:

O Driver Program coordena a aplicação → Cluster Manager aloca recursos → Worker Nodes executam as tarefas em paralelo → Resultados retornam ao Driver

Relação entre Spark Session e as Aplicações

🔑 Spark Session

A SparkSession é o ponto de entrada unificado para programar com Spark. Ela encapsula o SparkContext, SQLContext e HiveContext, fornecendo uma interface única para trabalhar com diferentes APIs do Spark.

📊 Acesso aos Dados

Interface unificada para ler dados de múltiplas fontes

⚙️ Configuração

Gerencia configurações da aplicação Spark

🗄️ SQL & DataFrames

Cria e manipula DataFrames e executa queries SQL

🔄 Contextos

Substitui SparkContext, SQLContext e HiveContext

Spark RDD

📦 Resilient Distributed Dataset

RDD é a estrutura de dados fundamental do Spark. É uma coleção imutável e distribuída de objetos que pode ser processada em paralelo.

🔒 Imutável

Uma vez criado, não pode ser modificado, apenas transformado

🌐 Distribuído

Dados particionados através do cluster

💪 Resiliente

Tolerante a falhas - reconstrói partições perdidas

⚡ Lazy Evaluation

Transformações só executam quando uma ação é chamada

Operações em RDD:

Transformações: map(), filter(), flatMap(), groupBy() - criam novos RDDs
Ações: count(), collect(), reduce(), save() - retornam valores ou gravam dados

Spark DataFrame

📊 DataFrame

DataFrame é uma coleção distribuída de dados organizada em colunas nomeadas, similar a uma tabela em banco de dados relacional ou um dataframe do Pandas.

🏗️ Estruturado

Dados organizados em schema com tipos definidos

🚀 Otimizado

Catalyst Optimizer para consultas eficientes

💡 Intuitivo

API similar ao Pandas e SQL

🔄 Interoperável

Pode ser convertido para RDD e vice-versa

Spark DataFrame vs RDD

Característica RDD DataFrame
Estrutura Não estruturado Estruturado com schema
Performance Boa Excelente (otimizado)
API Funcional (map, filter) SQL-like e funcional
Otimização Manual Automática (Catalyst)
Uso Recomendado Dados não estruturados Dados estruturados

💡 Recomendação: Use DataFrames sempre que possível para melhor performance e produtividade. Use RDDs apenas quando precisar de controle fino sobre operações ou trabalhar com dados não estruturados.
